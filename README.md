# CRA Evidence OS

OpenAI Build Week **Work & Productivity** entry: a traceable evidence workflow for EU Cyber Resilience Act preparation. It turns synthetic product documents and SBOMs into grounded GPT-5.6 suggestions, explicit human approvals, a change-aware Evidence Graph, incident worksheets, and a draft Annex VII package.

> Evidence readiness is not legal compliance. AI never approves claims or submits incidents.

## Build Week slice

This repository started from commit `e46828b` with only a placeholder README. The contest slice is the complete application history after that point. The post-contest roadmap is intentionally not presented as implemented.

## Run locally

Requires Node.js 22+ and pnpm.

```bash
pnpm install
copy .env.example .env.local
pnpm dev
```

Set `DEMO_FIXTURE_MODE=true` for the clearly labelled, synthetic local workflow. Real model calls require `OPENAI_API_KEY`; the configured model is `gpt-5.6-sol`, Responses API, Structured Outputs, reasoning effort `medium`, and `store: false`.

## GPT-5.6 contribution

1. Extract grounded claim suggestions with source fragment IDs.
2. Explain deterministic SBOM blast radius and prioritize review.
3. Draft 24h/72h incident fields only from approved facts, listing missing information.

Every output is schema-validated and remains `SUGGESTED`; application code, not the model, determines stale status. See `docs/CODEX_AND_GPT56.md` and the eval report.

The Evidence Inbox accepts PDF or JSON up to 5 MB, verifies file signatures, hashes bytes, stores blobs in R2 and tenant-scoped metadata/audit events in D1. The Dossier screen downloads a physical PDF and ZIP manifest package generated by `scripts/build-demo-export.py`.

## Security boundaries

Synthetic data only. Documents are untrusted input. Tenant scope is server-derived. No arbitrary URL fetch, code execution, automatic ENISA submission, or hidden fallback from live AI to fixtures. Production storage uses D1 metadata and R2 blobs when deployed; deployment is outside the locally authorized scope.
